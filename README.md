# ⚖️ AI 法律訴訟書狀自動化套件 (Legal Pleading AI Suite)

本開源套件專為台灣法律實務工作者（律師、法務）設計，透過整合 AI Agent 與 Google NotebookLM 的檢索能力，實現從「卷宗/判決見解分析」到「自動生成符合台灣法院格式之 Word/ODT 訴訟書狀與爭點整理表」的全自動化與標準化工作流。

## 📦 套件包含的技能模組 (Included Skills)

本專案強烈建議以單一庫（Monorepo）形式搭配使用，完整發揮 AI 自動化接力排版的威力。全套件包含以下四個核心模組：

### 1. `draft-pleading`（核心撰狀引擎）
* **功能**：接收使用者的訴訟資料，分析爭點並跨庫查詢 NotebookLM。採用 CRAC（結論-規則-應用-結論）論證格式，產出結構嚴謹的法律書狀 Markdown 草稿，並在使用者確認後，**預設自動生成最終的 DOCX (Word) 檔**。
* **特色**：內建公私配置分離，預設連線至開源版知識庫，使用者亦可一鍵切換至私有 NotebookLM 筆記本。

### 2. `pleading-table`（表格與證據清單排版引擎）
* **功能**：專門處理複雜的 Word 表格繪製與精確對齊。
* **可生成的三種表格**：
  1. **爭點整理表**（必出於爭點整理狀）：將事實上與法律上爭點合併為單一規格表格。
  2. **聲請調查證據表**（選出）：若草稿中含調查證據聲請，將自動產生專用表格。
  3. **高院證據清單表**（獨立附件）：當特別指定時，系統會自動產出一份獨立的 DOCX 附件檔。

### 3. `docx-to-odt`（ODT 轉檔與版面修正機制）
* **功能**：透過 LibreOffice CLI 將 DOCX 轉換為極度標準化的 ODT 格式，自動關閉「中文標點符號懸尾（Hanging Punctuation）」並將行編號調整至頁面外側。

### 4. `notebooklm-skill`（知識庫連線模組）
* **功能**：允許 AI 代理程式在背景高速查詢您的 Google NotebookLM 筆記本。本專案整合了由開源社群（@PleasePrompto）開發的模組，作為本套件的「案件記憶大腦」。

## 🚀 核心工作流程與快速開始 (Quick Start)

安裝完畢各項技能後，您只需在對話中給予明確指令，例如：

> 「請幫我寫一份民事起訴狀，被告是ＯＯ企業，請求給付工資。請去 NotebookLM 幫我查詢近期的最高法院見解，並請附上證據清單。」

**AI 自動接力流程**：
1. **讀取資料**：AI 自動遍歷工作目錄，讀取案情與卷證檔案。
2. **制定計畫**：分析攻防方向、擬定證據鏈，並產出撰寫計畫供使用者審閱。
3. **HyDE 查詢**：背景生成敵我假說，自動呼叫 `notebooklm-skill` 查詢見解。
4. **生成草稿**：呼叫 `draft-pleading` 產出格式精確的 Markdown 訴狀草稿，供使用者作文字確認。
5. **最終排版**：草稿確認後，預設自動呼叫內部排版引擎生成 `.docx` (Word) 檔。

### 💡 進階觸發指令 (Advanced Triggers)

本套件之部分進階功能，需仰賴使用者在提示詞中下達特定標記才會啟動：

* **觸發「爭點整理表」**：
  在撰寫爭點整理狀時，必須**明確指令要求**撰寫「附表格的爭點整理狀」或「含爭點整理表」才會觸發（若未提及則維持純文字排版）。產出的書狀將包含「爭點整理表」；若有附帶調查證據聲請，亦會自動在表格末加附「聲請調查證據表」。
* **觸發「高院證據清單表附件」**：
  不限於爭點整理狀，當明確要求「請附上高院證據清單」、「高院格式證據清單」時，只要卷宗包含證據資訊，系統都會在主書狀之外，**自動額外產出一份獨立的高院 7 欄格式 DOCX 附件檔**。
* **觸發「ODT 轉檔」**：
  系統預設僅會產出 DOCX。**只有特別要求**（例如：「寫一份起訴狀，最後請生成 ODT 檔」，或在 DOCX 生成後追加指令：「請幫我轉成 ODT」）時，系統才會觸發中文排版與標點符號懸尾修正，最終產出 ODT。

## 📂 最佳實踐：工作區與檔案準備 (Best Practices)

為了讓 AI 達到 100% 的精準度與極致效率，**強烈建議您採用以下標準工作流**：

### 📌 1. 一案一資料夾（案卷隔離）
強烈建議**「每接一個訴訟案件，就建立一個獨立的電腦資料夾」**（作為工作區），並將 AI 終端機執行於該目錄下。這能防止 AI 混淆不同案件的文檔，並能準確延續上一份書狀的證據編號與既有卷宗頁碼。

### 📌 2. 準備「案件基本資料.txt」
開始撰狀前，強烈建議於工作資料夾下儲存一份 `案件基本資料.txt`，內容應包含：
* **訴訟當事人對稱**（如：原告、被告、上訴人）
* **姓名或名稱**
* **地址**
* **電話與電子郵件信箱**
* **法院名稱**與股別案號（若有）
* **案由**（如：給付工資，這將決定書狀首行的動詞）

*AI 啟動時會自動掃描此檔，為您零失誤地自動生成「狀首」、「狀尾」與「法院公鑒」等格式欄位。*

### 🎙️ 3. 委任人訪談與錄音逐字稿協作
律師可以先與委任人進行案情討論，並全程錄音後轉譯成逐字稿（存為文字檔）放入工作區中。在使用撰狀技能時，您可以直接要求 AI：**「請參考目錄下的『會談逐字稿.md』來撰寫書狀。」** AI 便會精準從當事人訪談脈絡中萃取事實經過與訴之聲明。

### 📄 4. 檔案轉檔建議：全面 Markdown (.md) 化
雖然系統支援讀取 `.pdf`、`.docx`、`.odt` 等文件，但**強烈建議使用者將所有卷證文檔（包含所有的證據、自己歷次書狀、對方書狀、開庭筆錄、法院裁判等）全部先行轉換為 Markdown (`.md`) 格式**，並存放在該案件資料夾中。

* **標準起手式**：要求 AI：「請先讀取本目錄下的所有 `.md` 卷證檔案後，再為我撰寫『起訴狀』/『答辯狀』/『上訴理由狀』。」
* **核心優勢**：Markdown 是一種輕量的純文字結構格式，能完全去除 Word/PDF 在轉換時產生的無效換行、隱藏表格、浮水印等雜訊。這不僅能大幅節省 Token，更能讓 AI 的讀取理解達到最精確，徹底防止遺漏關鍵證據細節。

### 📓 5. 專屬 NotebookLM 筆記本資料建檔 (直接匯入雲端)
**除了在本地讀取檔案外，使用者也可以將案件的「所有資料」直接上傳到 NotebookLM 內的一個特定筆記本。** 在撰狀指令中，您可以先要求 AI：「請先去讀取我指定的 NotebookLM 筆記本的全部資料，再為我寫書狀。」這能極大化利用雲端大模型來梳理與記憶龐大的訴訟案情。

## 📝 書狀特有功能 (Pleading Features)
* **證據或文件清單彈性生成**：生成書狀時，預設會使用純文字模式輸出（例如：`甲證1：XXX契約書`）。若案件證卷繁雜，您可在指令中明確要求產生「證據或文件清單表格」，系統便會轉為生成結構化的 Word 三欄表格。
* **自動判斷狀首動詞**：系統會依據您要求的書狀名稱（起訴狀、上訴理由（二）狀等）與案由，自動為首行填入精確的「起訴」、「補充上訴理由」、「再提答辯」等法院標準動詞用語。
* **狀首日期自動生成 (選用)**：系統預設不顯示狀首日期。若您有此需求，可直接對 AI 說：「**以後撰寫書狀請幫我在狀首加上日期**」。AI 會記錄您的偏好，未來產生的所有書狀都會自動在名稱右下方生成一個「靠右對齊、12 點字」的日期。

## 🎨 法律結構可視化 (Legal Architecture)

本套件整合了專業的法律繪圖引擎，能將複雜的法律關係自動轉化為美觀、專業的 HTML 可視化圖表。

### 使用場景
您可以要求 AI 針對您的案件執行以下動作：
* **「畫出本案當事人關係圖」**：釐清原被告、第三人、代理人之間的法律關係。
* **「生成證據鏈分析圖」**：將證據、待證事實與法律依據進行邏輯連結。
* **「請求權基礎結構圖」**：可視化呈現民法第 184 條等複雜的構成要件涵攝過程。
* **「訴訟程序流程圖」**：針對上訴、調查證據等程序進行視覺化呈現。

### 改進與特色
* **適合列印與嵌入**：生成的圖表為單一 HTML 檔案，內嵌 SVG，採用高品質的淺色主題，適合直接列印或截圖嵌入 Word/PPT 中。
* **中文優化**：針對中文長字串自動斷行處理，防止文字溢出。
* **語意配色**：自動根據節點類型（當事人、證據、法律規範、裁判結果）套用標準化的專業配色。

### 致謝與出處
本功能之繪圖引擎 `legal-architecture` 由 **JeeC** 參考 **Cocoon AI** 的 `architecture-diagram` 專案開發並授權使用。我們在此向原作者的開源貢獻致以誠摯謝意，並將其法律化的二次創作發揚光大。

## 🛠️ 進階工具：ODT 互動巨集 (Tab 升降級)

LibreOffice Writer 預設並不支援像 Word 一樣「按下 Tab 鍵同時改變清單層級與段落樣式」的功能。為了解決這個問題，本套件提供了一組 **LibreOffice 互動巨集**。

### 功能特色

安裝此巨集後，當您在 LibreOffice Writer 中編輯由本套件生成的 ODT 檔案時：
* 在 `通用_層級1~4` 段落開頭按 **`Tab`**：會自動降一級（例如從 `通用_層級1` 變成 `通用_層級2`），並同步更新清單編號。
* 在段落開頭按 **`Shift+Tab`**：會自動升一級。
* 若游標在段落中間或非目標段落：維持 LibreOffice 原生行為（插入 Tab 字元）。

### 如何安裝

由於此功能會修改您電腦上 LibreOffice 的全域快捷鍵設定，因此系統**預設不會主動安裝**。您有兩種方式可以啟用它：

1. **直接請 AI 安裝**：
   在對話中直接向 AI 說：「**請幫我安裝 LibreOffice 的 Tab 升降級巨集**」。AI 會向您確認後自動在背景執行安裝。
2. **手動執行終端機指令**：
   您也可以自行在終端機中執行以下指令來完成安裝：
   ```powershell
   python "c:\Users\lex\.gemini\antigravity\skills\docx-to-odt\scripts\convert_docx_to_odt.py" --install-macros
   ```

*安裝完成後，請務必 **重新啟動 LibreOffice Writer** 才能使快捷鍵生效。*

## 📖 實例示範：清償借款起訴狀 (Example)

為了讓您更清楚了解套件的運作方式，以下以案情單純的「清償借款」為例，示範從準備檔案到產出書狀的完整流程。

### 步驟 1：準備工作與下達指令

**情境假設**：
債權人王大明借給債務人李小華新台幣 50 萬元，雙方簽有借據。約定於民國 112 年 12 月 31 日清償，利息以年息 5% 計算。清償期屆至後，李小華遲不還款，王大明決定向法院提起民事訴訟請求清償借款。

**填寫 `案件基本資料.txt`**：
請在您的工作資料夾中建立 `案件基本資料.txt`，並填入以下內容：

```text
原告：王大明
地址：台北市中正區重慶南路一段122號
電話：0912-345-678

被告：李小華
地址：新北市板橋區中山路一段161號

法院：臺灣新北地方法院
案由：清償借款
```

**觸發撰狀指令**：
檔案準備好後，您可以在對話中下達以下指令觸發撰狀技能：

> 「依據以下案情幫我撰寫一份民事起訴狀：原告借給被告 50 萬元，有借據為憑，約定 112 年 12 月 31 日清償，利息年息 5%。現在被告不還錢，請幫我寫起訴狀請求清償本金及利息。」

### 步驟 2：AI 協作與生成流程

下達指令後，`draft-pleading` 技能會依照以下互動流程與您協作：

1. **提出撰寫計畫**：技能會先分析案情並提出一份撰寫計畫給您審閱，確保訴之聲明與法理依據符合您的預期。您可以隨時提出修改意見。
2. **深度查詢知識庫（如有需要）**：如果是涉及「勞動爭議」等案件，技能預設會在提出撰寫計畫時自動擬定法律問題，並在下一個步驟查詢 NotebookLM 筆記本（預設為內建的勞動法知識庫，使用者有需要也可以另行建置私有筆記本），以實務見解補強書狀論點。
3. **生成草稿**：待您確認撰寫計畫後，技能便會自動生成結構嚴謹的訴訟書狀草稿（Markdown 格式）。
4. **草稿編修**：您可以審閱這份草稿，若有需要增刪調整的段落，可直接要求 AI 修改，直到您滿意為止。
5. **排版為最終書狀**：最後確認草稿無誤後，技能會自動接力將其排版，生成最終可用於遞狀的 Word (`.docx`) 檔案。

以下是經過前述流程後，技能最終嚴格依照法院書狀格式生成的 Markdown 草稿範例。此草稿將會被直接轉換為排版精美的 Word 檔：
