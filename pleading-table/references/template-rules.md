# 模板使用規則 — table-tmpl.docx

## 模板來源

`table-tmpl.docx` 由使用者直接提供，放置於 `assets/table-tmpl.docx`。  
模板不需重建，但其樣式與藍圖表格必須符合本文件規格。

## 一般書狀樣式

以下樣式與 `outline-docx` 共用：

| 樣式名稱 | 用途 |
|----------|------|
| `書狀_標題` | 書狀名稱（民事爭點整理狀） |
| `書狀_狀首日期` | 狀首日期行 |
| `書狀_預設` | 一般內文、法院名稱 + 公鑒、一般說明文字 |
| `書狀_謹狀` | 「謹狀」段落 |
| `書狀_簽章` | 具狀人、法定代理人、訴訟代理人、日期等 |
| `通用_層級1` | 第一層：`一、`、`二、`、`三、`、`四、` |
| `通用_層級2` | 第二層：`（一）`、`（二）`、`（三）` |

> [!IMPORTANT]
> `書狀_謹狀法院` 已刪除。
> 「法院名稱 + 公鑒」一律套用 `書狀_預設`，不得再引用不存在的 `書狀_謹狀法院`。

## 爭點表專屬樣式

| 樣式名稱 | 用途 | 套用位置 |
|----------|------|---------|
| `爭點表_標題` | 表頭欄位名稱 | 各表格表頭、欄位標題 |
| `爭點表_內容` | 表格一般內文 | 爭點欄、聲請調查證據表資料內容 |
| `爭點表_清單` | 爭點整理表三個清單欄 | 原因事實 / 法律依據 / 證據 |

> [!IMPORTANT]
> 舊名稱 `爭點表_列表` 已廢止。
> 所有程式、規格與草稿說明都必須改為 `爭點表_清單`。

## 固定中文編號段落規格

以下段落使用 `通用_層級1` 或 `通用_層級2` + numPr 設定。  
段落文字本身**不含**前綴，前綴由 Word numbering 自動產生。

### 第一層段落

| 最終顯示 | Word 樣式 | ilvl | 段落文字（不含前綴） |
|---------|----------|------|-------------------|
| 一、聲明：... | `通用_層級1` | 0 | `聲明：...` |
| 一、聲明： | `通用_層級1` | 0 | `聲明：` |
| 二、不爭執事項：... | `通用_層級1` | 0 | `不爭執事項：...` |
| 二、不爭執事項： | `通用_層級1` | 0 | `不爭執事項：` |
| 三、…爭點整理表 | `通用_層級1` | 0 | `…爭點整理表` |
| 四、聲請調查證據表 | `通用_層級1` | 0 | `聲請調查證據表` |

### 第二層段落

| 最終顯示 | Word 樣式 | ilvl | 段落文字（不含前綴） |
|---------|----------|------|-------------------|
| （一）... | `通用_層級2` | 1 | `...` |
| （二）... | `通用_層級2` | 1 | `...` |
| （三）... | `通用_層級2` | 1 | `...` |

> [!IMPORTANT]
> 聲明與不爭執事項都支援兩種模式：
>
> 1. 單項模式：以一個 `通用_層級1` 段落直接寫成 `聲明：...` 或 `不爭執事項：...`
> 2. 多項模式：先輸出一個 `通用_層級1` 段落 `聲明：` / `不爭執事項：`，再以 `通用_層級2` 逐項列出

> [!CAUTION]
> 絕對禁止：
> - 用 `書狀_預設` 硬寫 `一、聲明：`
> - 文字中手寫 `一、` / `（一）` 後又套用 `通用_層級1` / `通用_層級2`
>
> 正確做法：
> - 套用 `通用_層級1` 或 `通用_層級2`
> - 以 `numPr(numId, ilvl)` 指定層級
> - 文字本身不含編號前綴

## 中文 numbering 注入

`table-tmpl.docx` 本身的 numbering 定義可能不符合中文格式。  
腳本在載入模板後、清空內容前，應自 `outline-docx/assets/outline-base.docx` 複製中文 abstractNum 注入，建立新的 numId，供本書狀固定編號段落使用。

## 文件主體順序

最終 DOCX 主體順序如下：

1. 狀首
2. `一、聲明：` 或 `一、聲明：...`
3. `（一）...`、`（二）...`（僅在多項聲明時輸出）
4. `二、不爭執事項：` 或 `二、不爭執事項：...`
5. `（一）...`、`（二）...`（僅在多項不爭執事項時輸出）
6. `三、{party_status}爭點整理表`
7. 爭點整理表
8. `四、聲請調查證據表`（僅在 `evidence_request.items` 非空時輸出）
9. 聲請調查證據表（欄位標題列 + 資料列）
10. 表格後補充論述（若有）
11. 狀尾

## 藍圖表格規格

最新版模板含 **2 個藍圖表格**。  
程式必須在清空模板主體前先鎖定其 `_tbl` XML：

1. `doc.tables[0]`：爭點整理表藍圖
2. `doc.tables[1]`：聲請調查證據表藍圖（欄位標題列 + 資料列）

> [!IMPORTANT]
> 聲請調查證據表僅剩單一實體表格。表格A（「聲請調查證據表」標題列與「提出人/日期列」）已從模板與程式中全面刪除。

## 共通儲存格操作規則

> [!IMPORTANT]
> 所有表格儲存格都必須：
>
> 1. 先刪除儲存格內全部既有段落
> 2. 再以 `cell.add_paragraph()` 全新建立每一段落
> 3. 以程式強制套用段落樣式
>
> 不得保留或重用 deep-copied 的舊段落。

## 爭點整理表規格

### 表格用途

爭點整理表為單一表格，合併呈現事實上爭點與法律上爭點。  

### 欄位

4 欄：

1. `爭點`
2. `{reason_header}`
3. `法律依據`
4. `證據`

### 樣式規則

| 位置 | 樣式 |
|------|------|
| 表頭列 | `爭點表_標題` |
| 第 1 欄第 1 段（如 `爭點一：`） | `爭點表_內容` |
| 第 1 欄第 2 段（描述） | `爭點表_內容` |
| 第 2~4 欄所有項目 | `爭點表_清單` |

### 清單格式規則

對於原因事實 / 法律依據 / 證據三欄：

- 只有 1 項：`•\t{text}`
- 2 項以上：`1.\t{text}`、`2.\t{text}`…
- 第一項與後續項目的建立方式完全相同，均須全新建立 paragraph

### 跨頁規格

| 列類型 | 規則 |
|--------|------|
| 表頭列 | 必須設 `tblHeader`（跟頁重複）+ `cantSplit`，避免拆頁 |
| 資料列 | 不設 `cantSplit`，允許跟頁分割 |

資料列必須顯式移除從表頭 deep copy 繼承的 `w:cantSplit` 與 `w:tblHeader`。

## 聲請調查證據表規格

聲請調查證據表為**單一實體表格**，第 1 列為欄位標題列，第 2 列起為資料列。

若 `evidence_request` 不存在、為 `null`、或 `items` 為空，則下列內容全部不得輸出：

- `四、聲請調查證據表`
- 聲請調查證據表

### 表格規格

#### 結構

聲請調查證據表 為 1 + N 列：

- 第 1 列：欄位標題列
- 第 2 列起：資料列，每筆 `evidence_request.items[]` 對應 1 列

#### 第 1 列：欄位標題列

由左至右共 6 欄：

1. `編號`
2. `所涉爭點`
3. `調查事項`
4. `調查對象`
5. `對象地址及聯絡方式`
6. `待證事實`

樣式規則：

- 所有欄標題套用 `爭點表_標題`
- 本列為唯一需要跨頁重複的欄位標題列
- 必須設 `tblHeader`
- 可設 `cantSplit`

特殊顯示規則：

- `編號` 須在同一儲存格中分成 2 行顯示
- `對象地址及聯絡方式` 須在同一儲存格中分成 2 行顯示為：
  - `對象地址`
  - `及聯絡方式`

#### 第 2 列起：資料列

每筆 `evidence_request.items[]` 產生一列。  
資料列全部套用 `爭點表_內容`，且不得繼承 `tblHeader` 或 `cantSplit`。

欄位規則如下：

| 欄位 | 規則 |
|------|------|
| `編號` | 由 builder 自動填入 `1`、`2`、`3`… |
| `所涉爭點` | 只填 `爭點一`、`爭點二`…；若涉及多個爭點，於同一儲存格分行 |
| `調查事項` | 依上游 JSON 的 `investigation_item` 輸出 |
| `調查對象` | 依上游 JSON 的 `target` 輸出 |
| `對象地址及聯絡方式` | 依上游 JSON 的 `target_address_contact` 輸出，可依 `\n` 分行 |
| `待證事實` | 依上游 JSON 的 `fact_to_prove` 輸出 |

## 表格插入順序

聲請調查證據表為單一表格，直接插入於 anchor 前。

最終順序為：

- `四、聲請調查證據表`
- 聲請調查證據表（欄位標題列 + 資料列）

## 狀尾規格

### keepLines / keepNext

從「謹狀」開始的所有狀尾段落，均須加入 `<w:keepLines/>` + `<w:keepNext/>`，確保不跨頁。

### 謹狀正規化

無論原始文字是否含空格，輸出時統一正規化為 `謹狀`，套用 `書狀_謹狀`。

### 法院行正規化

法院全名中不得插入多餘空格；`公鑒` 前保留一個全形空白（`　`）。  
法院名稱 + 公鑒 一律套用 `書狀_預設`。
