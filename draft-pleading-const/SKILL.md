## 核心定位與工作流
本技能為純粹的**收斂與排版引擎**。假設使用者已經透過先前的對話或研究，產出了憲法違憲論點的大綱。

**步驟一：整理與輸出 Markdown 草稿**
請將使用者的論點素材，嚴格套入以下的 Markdown 骨架。

**步驟二：請求確認**
請將產出的 Markdown 草稿展示給使用者，並詢問是否同意。

**步驟三：編譯 Word 檔**
在使用者同意後，請**務必**執行終端機指令以編譯 Word 檔：
```bash
python "C:/Users/lex/.gemini/config/skills/draft-pleading/scripts/build_pleading.py" "草稿的絕對路徑" --template "C:/Users/lex/.gemini/config/skills/draft-pleading/assets/pleading-tmpl.docx"
```
（請注意：雖然這是憲法聲請書，但底層排版引擎與樣板皆共用於 `draft-pleading` 技能中，請勿更改路徑）

## 憲法聲請書 Markdown 骨架標準

請嚴格遵守以下格式，特別注意：
1. **當事人欄位**：「姓名/名稱」與「身分證明文件字號：」必須在同一行，地址必須換行。
2. **標題字眼**：請精準使用 `聲請審查客體`、`應受判決事項之聲明`、`主要爭點`、`聲請理由` 等字眼。
3. **引用檢查**：若提及「系爭規定」、「系爭判決」、「確定終局裁判」，或引用「釋字」、「憲判字」、「德國聯邦憲法法院裁判」等段落時，必須以 `「...」` 引號精準包覆原文，否則排版將會報錯中斷！
4. **附件清單**：清單標題為 `附件：`，下方項目使用 `附件1：`、`附件2：`。

```markdown
# 裁判及法規範憲法審查聲請書

聲請人 [姓名]　身分證明文件字號：[字號]
　　住：[地址]
　　電話：[電話]
　　送達代收人：[姓名]
　　送達處所：[地址]

代表人 [姓名]　身分證明文件字號：[字號]
　　住：[地址]

　　事務所：[地址]
　　電話：[電話]

茲依憲法訴訟法第59條之規定，聲請裁判及法規範憲法審查事：

**　　聲請審查客體**
（內容...）

**　　應受判決事項之聲明**
（內容...）

**　　主要爭點**
（內容...）

**壹、本件聲請符合受理要件**
一、原因案件事實及所經過之訴訟程序
(一) ...

**貳、應受審查客體之違憲理由**

附件：
附件1：用盡審級救濟之最終裁判及其送達日期之證明

具狀人：[姓名]
```
